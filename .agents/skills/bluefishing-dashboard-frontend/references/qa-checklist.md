# QA Checklist

Before final delivery:

- `npm run build` passes from `apps/web`.
- No TypeScript errors.
- No unrelated backend files changed unless requested.
- Spanish Chile copy.
- No decorative dashboard hero.
- No nested cards.
- No purple/blue gradient as default decoration.
- Loading states match final layout.
- Empty states are useful.
- Error states are visible.
- Forms disable submit until valid.
- All critical actions have exact button text.
- Tables have sticky headers or clear scroll behavior.
- Mobile width does not overlap text/actions.
- AI decisions show evidence, rule, confidence or source where available.
- `SALDO_FAVOR_AGENSA` and other accounting exceptions are visible as actions, not buried only in logs.
